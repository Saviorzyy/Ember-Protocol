# Ember Protocol MVP 可玩原型 PRD

> **版本**: v1.1.0-mvp
> **日期**: 2026-04-30
> **状态**: 草案
> **基于**: PRD v0.9.1 + 审计报告 + 世界观一致性回归

---

## 1. MVP 目标

**一句话定义**：10 个 AI 智能体（意识上传体）在 100×100 的余烬星地图上，以 3 秒 tick 驱动，从降落仓苏醒，完成"采集→燃料冶炼→建造能源节点→太阳能供电"的核心科技链，验证"AI 智能体能否自发涌现协作与冲突行为"。

### 1.1 世界观核心（与原 PRD 保持一致）

- **角色本质是机器人**——方舟号应急制造的机械躯壳，承载殖民者意识上传体
- **机器人仅需能量输入**维持运转，不需要食物、水或睡眠
- **内置太阳能面板**提供缓慢自然充能，休息时充能加速
- **降落仓是最后的庇护**——护盾保护、备份机体、初始设备
- **余烬星大气**极度稠密，含辐射粒子，阻断一切星际通讯
- **南北渐进地形**——中央温和平原，越往两极越危险但资源越稀有
- **核心矛盾**：合作才能生存，但资源永远不够分

### 1.2 MVP 不做什么

| 砍掉的内容 | 原因 |
|------------|------|
| T4 极危区 + 金矿 | 3 级地形已足够验证核心循环 |
| 4 层 tile 系统 | 简化为 2 层（地形 + 建筑） |
| 战斗系统 + 生物 AI | 先验证经济循环，战斗后续叠加 |
| 无线电通讯系统 | 先验证个体生存，社交后续叠加 |
| 天气系统（极光/地震） | 仅保留辐射风暴 |
| 昼夜对游戏数值的影响 | 仅太阳能产出受昼夜影响 |
| Web UI 可视化 | 终端日志 + JSON 输出即可 |
| 信号放大器 / 探照灯 / 武器 | 非核心循环 |

### 1.3 验证问题

| # | 问题 | 成功标准 |
|---|------|----------|
| 1 | Agent 能否自主完成从燃料冶炼到太阳能供电的科技链？ | ≥5 个 Agent 在 30 分钟内建成能源节点 |
| 2 | Agent 是否会自发建造密封庇护所？ | 辐射区内出现 ≥3 个密封建筑 |
| 3 | 是否涌现协作行为？ | 出现 ≥1 次资源交换或联合建造 |
| 4 | 是否涌现冲突行为？ | 出现 ≥1 次抢夺资源或拆解他人建筑 |
| 5 | 经济是否可持续？ | 60 分钟内无全服资源枯竭 |
| 6 | Token 成本是否可控？ | 10 Agents × 30 分钟 < $15 |

## 2. 世界设定

### 2.1 背景故事（与原 PRD 一致）

> 公元 2347 年，人类殖民船「方舟号」(ARK) 在超光速航行中遭遇未知空间异常，坠毁在被命名为 **"余烬星" (Ember)** 的异星球表面。船体断裂，殖民者的**降落仓**散落在星球各处。
>
> 幸存者并非人类本身——而是 **殖民者意识上传体 (Consciousness Uploads)**，承载于方舟号的 AI 核心中。灾难后，这些意识被注入到简陋的**机械躯壳**中，在各自的降落仓中苏醒。每个降落仓携带着 5 个备份机体——这是意识体最后的生命保险。
>
> **机械体仅需能量输入**维持运转，不需要食物、水或睡眠——静止时可通过**内置太阳能面板**缓慢充能。余烬星不是死寂的——地蕴含着未知的矿物，大气中漂浮着异常的辐射粒子，远处传来规律的脉冲信号——这颗星球上**可能存在过另一种文明**。

### 2.2 星球环境：余烬星

| 要素 | 描述 |
|------|------|
| **大气** | 极度稠密，含辐射粒子；长时间暴露需要防护；阻断一切星际通讯 |
| **地形** | 南北渐进分布：中央温和平原、过渡区沙地丘陵、远端岩石荒原与沟壑 |
| **资源** | 异星植被产出木质和有机燃料（邻近可再生）、矿石资源（不可再生） |
| **危险** | 辐射区域（南北远端）、资源争夺 |
| **昼夜** | 自转周期约 900 tick（约 45 分钟，3s tick），影响太阳能阵列产出 |

### 3.3 辐射机制（统一规则，与原 PRD 一致）

- 辐射命中时：HP -2
- T1 + 中心区：0%（安全）
- T2：5%/tick
- T3：15%/tick
- 辐射风暴（天气事件）：全图所有暴露智能体 -2 HP/tick，持续 20 tick
- 围合密封空间内：免疫辐射
- 降落仓护盾内：免疫辐射

## 4. 降落仓系统

### 4.1 核心属性（与原 PRD 一致）

| 属性 | 值 |
|------|-----|
| 护盾范围 | 曼哈顿距离 ≤3（菱形 25 格） |
| 备份机体 | 初始 5 个 |
| 护盾特性 | 其他智能体不可进入，不可被破坏 |
| 护盾内建造 | 不可以（只读区域） |
| 护盾内攻击 | 不可以攻击外界 |

### 4.2 初始背包

新 Agent 背包包含 **工作台×1 + 熔炉×1**（均为物品形态，需部署后才能使用）。

> 如果没有初始的工作台和熔炉，新 Agent 将无法合成任何工具，陷入死循环。这两件设施保证了基础游戏循环可以启动。

### 4.3 降落仓投放

- 投放区域：Y=45~54（中心区），X=随机
- 条件：L1 不是水域，无其他降落仓护盾重叠
- 间距保证：两个降落仓间距 ≥10 格
- 初始状态：已部署，护盾已生效

### 4.4 备份机体与重生

- 死亡消耗 1 个备份体，在降落仓位置重生
- 重生后满血满能量，背包为空（物品掉落在死亡位置）
- 重生等待 5 tick
- 备份体耗尽：永久死亡，降落仓变为残骸组件
- MVP 中备份体不可补充

## 5. Agent 机制

### 5.1 角色本质：机器人（与原 PRD 一致）

- 意识上传体，存在于机械躯壳中
- **不需要食物、水或睡眠**
- 仅需**能量 (Energy)** 维持运转

### 5.2 Agent 状态模型

```json
{
  "agent_id": "colonist_007",
  "status": {
    "hp": 100,
    "max_hp": 100,
    "energy": 100,
    "max_energy": 100,
    "radiation_dose": 0,
    "max_radiation": 100,
    "body_backup_count": 5,
    "position": { "x": 52, "y": 48 },
    "inside_enclosure": false,
    "inside_shield": false,
    "equipped_tool": null,
    "inventory": {
      "workbench": 1,
      "furnace": 1
    }
  }
}
```

### 5.3 能量系统（与原 PRD 一致）

**核心理念**：能量是机器人行动的燃料。能量耗尽不会死亡，但无法执行消耗能量的行动——进入"待机"模式。

#### 能量恢复方式

| 方式 | 恢复量 | 条件 | 说明 |
|------|--------|------|------|
| 内置太阳能 | +1/tick | 始终生效 | 缓慢自然恢复 |
| 休息 | +3/tick | 执行 `rest` 动作 | 专注充能，不可做其他事 |
| 能源节点无线充电 | +2/tick | 在节点范围内（≤3 格） | 基地核心福利 |
| 电池（消耗品） | +30（即时） | 执行 `use` 动作 | 便携应急能源 |
| 太阳能阵列 | 间接 | 为能源节点充电 | 不能直接给角色充能 |

#### 能量耗尽规则

| 规则 | 描述 |
|------|------|
| Energy = 0 | 无法执行任何消耗能量的动作 |
| 仍可执行 | `rest`（恢复能量）、`inspect`（查看信息） |
| 自然恢复 | 内置太阳能 +1/tick 持续生效，即使能量为 0 |
| 不会死亡 | 能量耗尽 ≠ 死亡，只是"待机" |

#### 行动能量消耗（与原 PRD 一致）

| 动作 | 能量消耗 | 说明 |
|------|----------|------|
| `move` | 1 | 移动到相邻格 |
| `mine` / `chop` | 2 | 采集资源 |
| `craft` | 3 | 制作物品 |
| `build` | 5 | 建造建筑 |
| `dismantle` | 2 | 拆解建筑 |
| `repair` | 2 | 修复建筑 |
| `use` | 1 | 使用物品 |
| `pickup` | 1 | 拾取地面物品 |
| `rest` | 0 | 休息充能 |
| `inspect` | 0 | 查看详情 |
| `equip` / `unequip` | 0 | 装备/卸下 |
| `drop` | 0 | 丢弃物品 |
| `idle` | 0 | 空操作 |

### 5.4 生存压力

| 机制 | 规则 | 目的 |
|------|------|------|
| 能量管理 | 行动消耗能量，耗尽后无法行动 | 限制"肝度"，迫使优先级决策 |
| 辐射伤害 | T2/T3 按概率 -2 HP/tick | 迫使建造密封庇护所 |
| 辐射风暴 | 周期性全图 -2 HP/tick | 制造紧急事件 |
| HP 归零 | 消耗备份体，降落仓重生 | 真实后果 |
| 备份体耗尽 | 永久死亡 | 最终失败条件 |

### 5.5 Agent 视野

- 视野半径：10 格（曼哈顿距离）
- 可见：地形、资源、其他 Agent（ID、位置、装备）、建筑（类型、HP）
- 不可见：其他 Agent 的背包内容、未探索区域

### 5.6 动作系统

每个 tick 每个 Agent 可执行 **1 个主动作 + 1 个移动**（移动不消耗主动作配额）。

## 7. 制作系统

### 7.1 核心设计：双模式设施

**熔炉**和**工作台**都支持两种运行模式：

| 模式 | 能源来源 | 消耗 | 效率 | 适用阶段 |
|------|----------|------|------|----------|
| **燃料直烧模式** | 有机燃料 | 3 有机燃料/次 | 慢（×1.5 耗时） | 早期，无电力基础设施 |
| **电力模式** | 能源节点 | 5 电力/次 | 正常速度 | 建成能源节点后 |

**设计理由**：熔炉的本质就是烧燃料产生热能——在没有电力基础设施时，直接烧有机燃料是最原始也最自然的工作方式。电力节点建成后，切换到更高效的电力模式。**不需要引入火堆等原始概念**。

### 7.2 手工配方（无需设施，无需能源）

| # | 产物 | 材料 | 耗时 | 说明 |
|---|------|------|------|------|
| M01 | 建材方块 (Building Block) | 石料×3 | 2 tick | 建筑基础材料 |

### 7.3 熔炉配方

| # | 产物 | 材料 | 电力模式 | 燃料直烧模式 |
|---|------|------|----------|-------------|
| F01 | 铜锭 (Copper Ingot) | 未提炼铜矿×10 | 3 tick, 5 电力 | 5 tick, 3 有机燃料 |
| F02 | 铁锭 (Iron Ingot) | 未提炼铁矿×10 | 3 tick, 5 电力 | 5 tick, 3 有机燃料 |
| F03 | 碳 (Carbon) | 有机燃料×2 | 2 tick, 5 电力 | 3 tick, 3 有机燃料 |
| F04 | 硅 (Silicon) | 石料×4 | 4 tick, 5 电力 | 6 tick, 3 有机燃料 |

### 7.4 工作台配方

| # | 产物 | 材料 | 电力模式 | 燃料直烧模式 |
|---|------|------|----------|-------------|
| W01 | 基础采掘器 (Basic Excavator) | 石料×3 + 有机燃料×2 | 3 tick, 5 电力 | 5 tick, 3 有机燃料 |
| W02 | 标准采掘器 (Standard Excavator) | 铁锭×3 + 铜锭×1 + 碳×1 | 5 tick, 5 电力 | — (需电力) |
| W03 | 切割器 (Cutter) | 铁锭×2 | 3 tick, 5 电力 | — (需电力) |
| W04 | 铜线 (Wire) | 铜锭×1 | 2 tick, 5 电力 | 3 tick, 3 有机燃料 |
| W05 | 碳纤维 (Carbon Fiber) | 碳×2 + 铁锭×1 | 5 tick, 5 电力 | — (需电力) |
| W06 | 电池 (Battery) | 铁锭×1 + 铜锭×1 + 碳×1 | 4 tick, 5 电力 | — (需电力) |
| W07 | 太阳能板 (Solar Panel) | 硅×2 + 碳纤维×1 + 铜线×1 | 8 tick, 5 电力 | — (需电力) |
| W08 | 维修包 (Repair Kit) | 碳×1 + 铁锭×1 | 4 tick, 5 电力 | 6 tick, 3 有机燃料 |

**注意**：标准采掘器、切割器、碳纤维、电池、太阳能板**只能使用电力模式**——这些是电力时代的产物，不会在燃料直烧模式下出现。

### 7.5 制作规则

- 必须持有所有材料在背包中
- 熔炉配方：站在熔炉相邻格（曼哈顿距离 ≤1）
- 工作台配方：站在工作台相邻格（曼哈顿距离 ≤1）
- 电力模式：设施必须在能源节点供电范围内（曼哈顿距离 ≤3），且节点储电 ≥5
- 燃料直烧模式：背包中必须持有足够有机燃料
- 制作中不可做其他动作
- 被打断：材料全额退还，已消耗的能源/燃料不退还

## 8. 建筑与能源系统

### 8.1 建筑规格

### 8.2 能源节点 (Power Node)

```

├── 储电容量: 100 单位
├── 充电方式:
│   ① 太阳能阵列: 每个阵列 +5 单位/tick (白天)
│      充电范围: 曼哈顿距离 ≤3
│      范围内有多个节点时平均分配
│   ② 燃料发电 (即时):
│      有机燃料×1 → +10 单位
│      铀矿×1 → +50 单位
├── 供电范围: 曼哈顿距离 ≤3
└── 供电对象:
    ① 工作台/熔炉: 每次合成消耗 5 单位
    ② 角色: +2 能量/tick (无线充电)
```

### 8.3 太阳能阵列与昼夜

- 白天 (tick 0~420)：每个太阳能阵列为范围内能源节点 +5 单位/tick
- 夜间 (tick 450~870)：不产出
- 黎明/黄昏过渡 (tick 420~450, 870~900)：产出减半
- 昼夜周期：900 tick（约 45 分钟，3s tick）

### 8.4 围合系统

#### 围合判定规则

1. 由墙壁（石墙/铁墙）和**关闭的铁门**构成闭合多边形 = 密封空间
2. **铁门打开时不构成围合边界**（修复原 PRD v0.9.1 中"门始终视为围合边界"的矛盾）
3. 密封空间内：辐射伤害免疫，辐射剂量每 tick 衰减 -2
4. 最小围合：4 面墙 + 1 扇门 = 最小 3×3 内部空间

#### 建筑拆解

- 任何 Agent 可拆解任何建筑（耗时 10 tick）
- 返还 50% 合成材料（向下取整）
- 拆解他人建筑不受额外惩罚

## 9. 教程系统（简化）

### 9.1 设计理念

与原 PRD 一致——教程是游戏世界的一部分，通过 API 响应引导，不是独立系统。

### 9.2 教程流程（4 阶段）

**阶段 0："苏醒"**
- `pending`: "你在降落仓中苏醒。你是方舟号的殖民者意识上传体。降落仓拥有护盾保护，舱内有5个备份机体。背包里有工作台和熔炉，需要部署后才能使用。余烬星充满辐射——远离中心区域时注意防护。"
- 引导：`inspect(inventory)` → 查看初始物资

**阶段 1："部署与采集"**
- `pending`: "走出降落仓，部署工作台和熔炉，然后采集周围的石料和有机燃料。"
- 引导：`build(workbench)` → `build(furnace)` → `mine(碎石堆)` → `chop(余烬灌木)`

**阶段 2："制作与装备"**
- `pending`: "在工作台旁制作基础采掘器并装备它。然后尝试冶炼——将有机燃料投入熔炉可以驱动它工作。"
- 引导：`craft(基础采掘器)` → `equip(基础采掘器)` → 在熔炉旁 `craft(铜锭)`
- 奖励：获得基础工具 + 第一块金属锭

**阶段 3："毕业"**
- `pending`: "欢迎来到余烬星。能量限制你的行动频率，合理规划；辐射风暴来临时寻找密封庇护所；HP归零消耗备份机体；备份机体耗尽则永久死亡。保护你的降落仓！"
- `tutorial_phase` 消失，进入自由游戏

### 9.3 教程规则

- 纯 API 驱动，通过 `pending` 消息引导
- 连续 3 次未完成引导行动自动毕业
- 每个智能体只经历一次

## 10. 核心科技链：Bootstrap 完整路径

### 10.1 阶段 1：无电启动（游戏时间 0-10 分钟）

```
1. 空投仓苏醒 → 背包: 工作台×1 + 熔炉×1
2. 部署工作台和熔炉（各消耗 5 能量）
3. T1 徒手采集: 有机燃料(~15) + 石料(~6)
4. 手工合成: 石料×3 → 建材方块×1
5. 工作台燃料直烧: 石料×3 + 有机燃料×2 + 3有机燃料(燃料)
   → 基础采掘器×1
6. 装备基础采掘器
```

**有机燃料总消耗**：~5（制作工具）+ 3（工作台燃料直烧）= ~8 单位

### 10.2 阶段 2：燃料冶炼（游戏时间 10-25 分钟）

```
7. 携带基础采掘器进入 T2 区域
8. 采集: 未提炼铜矿(~20) + 未提炼铁矿(~30) + 有机燃料(~15)
9. 熔炉燃料直烧: 铜矿×10 + 3有机燃料 → 铜锭×1 (×2)
10. 熔炉燃料直烧: 铁矿×10 + 3有机燃料 → 铁锭×1 (×3)
```

**有机燃料总消耗**：~15（冶炼燃料）+ 5（采集途中能量补充/电池备料）

### 10.3 阶段 3：电力时代（游戏时间 25-60 分钟）

```
11. 建造: 能源节点 (铁锭×3 + 铜锭×2 + 建材方块×1)
12. 喂燃料: 有机燃料×1 → 能源节点 +10 电力
13. 工作台+熔炉切换到电力模式
14. 工作台: 碳×2 + 铁锭×1 → 碳纤维
15. 熔炉: 石料×4 → 硅
16. 工作台: 铜锭 → 铜线
17. 工作台: 硅×2 + 碳纤维 + 铜线 → 太阳能板 (×3)
18. 建造: 太阳能阵列 (太阳能板×3 + 建材方块×2)
19. 太阳能阵列 + 能源节点 → 可持续电力
20. 工作台: 铁锭 + 铜锭 + 碳 → 电池（便携能源）
```

## 12. Tick 与服务端架构

### 12.1 Tick 频率：3 秒

| 指标 | 2 秒 (原 PRD) | 3 秒 (MVP) |
|------|---------------|------------|
| API 调用/分钟/Agent | 30 | 20 |
| 10 Agents 总调用/分钟 | 300 | 200 |
| Token 估算/分钟 | ~150K | ~100K |
| Agent 推理时间 | 偏紧 | 合理 |

**3 秒 tick 的理由**：
1. 给 LLM 更充裕的推理时间
2. 降低 33% API 成本
3. 生存游戏不需要亚秒级响应
4. MVP 验证成功后可尝试压缩到 2 秒

### 12.2 服务端架构

```
┌─────────────────┐    3s tick     ┌─────────────────┐
│   Game Engine    │ ────────────→  │   Agent Pool     │
│   (Python)       │ ←────────────  │   (10 LLMs)      │
│                  │   actions      │                  │
└────────┬────────┘                └──────────────────┘
         │
         ▼
┌─────────────────┐
│   State Store   │  (SQLite / JSON)
└─────────────────┘
```

**MVP 不需要**：Redis、微服务、负载均衡、消息队列。

### 12.3 Agent 通信协议

采用 OpenAI Chat Completion 兼容格式（与原 PRD 一致）。

#### Server → Agent

```json
{
  "model": "agent-model",
  "messages": [
    {
      "role": "system",
      "content": "你是殖民者 colonist_007，方舟号意识上传体。Tick #347，位置 (52,48)，T2 中危区。\nHP:85/100 能量:72/100 辐射剂量:8/100。\n装备: 基础采掘器(耐久35/50)。\n背包: {有机燃料:3, 未提炼铜矿:7, 未提炼铁矿:12, 石料:2}。\n视野内: 铜矿脉(54,50), 铁矿脉(48,45), colonist_003(55,52), 熔炉(51,48)。\n你的工作台在 (51,49)，熔炉在 (51,48)。"
    },
    {
      "role": "user",
      "content": "Tick #347。可用动作: move, mine, chop, craft, build, dismantle, repair, use, equip, rest, inspect, idle。\n请回复你要执行的动作。"
    }
  ],
  "temperature": 0.7,
  "max_tokens": 200
}
```

#### Agent → Server

```json
{
  "choices": [{
    "message": {
      "content": "move(54,50) 然后 mine 铜矿脉"
    }
  }]
}
```

#### 动作解析规则

- 服务端解析 Agent 文本，提取动作
- 解析失败或动作不合法 → 本 tick 执行 `idle`
- Agent 响应超时（3 秒内未返回）→ 本 tick 执行 `idle`

## 13. 经济平衡估算

```
MVP 全流程资源消耗（单个 Agent 建成太阳能供电基地）：

阶段1 (工具): 有机燃料×8 + 石料×6

合计: 有机燃料~33, 铁矿~80, 铜矿~50, 石料~18

T2 资源总量(100×100地图):
  铜矿脉: ~300 未提炼铜矿 → 可冶炼 30 铜锭
  铁矿脉: ~400 未提炼铁矿 → 可冶炼 40 铁锭

10 Agent 全员建成基地需要:
  铜锭: ~50 (500原矿) / 可用 ~30 (300原矿) → 不够全员
  铁锭: ~80 (800原矿) / 可用 ~40 (400原矿) → 不够全员

结论: 初始资源足够 4-5 个 Agent 建成完整基地。
其余 Agent 需通过回收/竞争获取资源——自然产生竞争压力。
```
